# 文件面板工作流设计

Feature Name: file-panel-workflow
Updated: 2026-07-17

## 设计

文件列表在获取后按目录、名称排序。上传入口分别配置文件选择和目录选择；目录上传读取选择器提供的相对路径，创建远端目录后复用传输任务管理器上传文件。上传弹窗订阅任务状态，后台按钮关闭弹窗并在终端状态栏持续显示进度。

文件面板通过组件本地全屏状态覆盖终端内容。编辑器监听 Capacitor App 的 `backButton` 事件，在编辑器打开时消费该事件并关闭编辑器。

删除目标保存在组件状态中。删除操作使用覆盖式确认弹窗，用户确认后复用现有删除请求并刷新文件列表。

## 验证

- 目录优先于文件与压缩包显示。
- 文件和目录选择器均可创建传输任务。
- 上传弹窗与状态栏显示任务进度。
- 全屏文件面板可返回终端。
- Android 返回键关闭编辑器。
