# 服务器列表性能与交互设计

Feature Name: server-list-performance
Updated: 2026-07-17

## 设计

指标采集将 CPU、内存、磁盘、温度、磁盘 I/O 和网络统计组合为单次 SSH 命令，使用唯一分隔标记拆分回传内容。采集队列通过 4 个工作任务限制并发度，并在单台服务器完成后合并指标状态，保留其他已完成服务器的历史结果。自动刷新在上一轮采集完成后使用延时任务安排下一轮，自动刷新间隔随已连接服务器数量扩展。

列表标题和底部服务器导航图标使用 350 毫秒双击检测，并调用滚动容器的平滑 `scrollTo` 方法。搜索框和筛选容器设置相同的固定高度。

## 验证

- 每台服务器的一次指标采集只调用一次远程命令。
- 连接 21 至 50 台服务器时自动刷新间隔为 30 秒。
- 连接超过 50 台服务器时自动刷新间隔为 60 秒。
- 双击标题后列表滚动至顶部。
