# 服务器列表性能与交互需求

## 需求

### 指标采集

当服务器列表采集指标时，系统应为每台已连接服务器使用一次远程命令获取列表所需指标。

当已连接服务器数量超过 20 台时，系统应每 30 秒发起一次自动指标刷新。

当已连接服务器数量超过 50 台时，系统应每 60 秒发起一次自动指标刷新。

系统应以最多 4 台服务器的并发度采集指标，并在每台服务器采集完成后显示该服务器的指标。

系统应在当前指标采集完成后安排下一次自动指标采集。

### 列表交互

当用户双击服务器列表标题或底部服务器导航图标时，系统应平滑滚动服务器列表至顶部。

搜索框和状态筛选控件应使用相同高度。

系统应显示“内存”和“磁盘”作为对应指标名称。
