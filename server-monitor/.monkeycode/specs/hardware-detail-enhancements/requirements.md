# 硬件详情增强需求

## 简介

服务器详情页展示 CPU 型号、磁盘 SMART 健康状态与磁盘温度。

## 术语

- **CPU 型号**：服务器内核公开的处理器标识。
- **磁盘健康状态**：SMART 工具报告的单块物理磁盘状态。
- **磁盘温度**：SMART 工具报告的单块物理磁盘温度，单位为摄氏度。

## 需求

### 需求 1：跨架构 CPU 型号

**用户故事：** 作为服务器管理员，我希望详情页展示 x86 和 ARM 服务器的 CPU 型号，以便识别硬件配置。

#### 验收标准

1. 当服务器公开 `model name` 字段时，系统应展示该字段的值。
2. 当服务器公开 ARM 处理器字段 `Hardware`、`Processor`、`cpu model` 或 `CPU part` 时，系统应展示首个可用字段的值。
3. 当处理器型号字段缺失时，系统应展示处理器架构标识。

### 需求 2：磁盘温度与健康状态

**用户故事：** 作为服务器管理员，我希望直接查看每块磁盘的温度和健康状态，以便及时识别磁盘风险。

#### 验收标准

1. 当 SMART 工具提供磁盘温度时，系统应在对应磁盘健康行展示摄氏温度。
2. 当 SMART 工具报告 `PASSED`、`OK` 或 `GOOD` 时，系统应以绿色状态徽标展示“正常”。
3. 当 SMART 工具报告其他健康结果时，系统应展示原始状态并使用警示色。
4. 当服务器未提供 SMART 工具或磁盘 SMART 数据时，系统应继续展示文件系统磁盘用量。
